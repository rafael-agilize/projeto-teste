  <name>Task 1: Build MatchManager and wire elimination + win logic</name>
  <files>src/server/multiplayer/matchManager.ts, src/server/handlers/gameHandlers.ts</files>
  <action>
Create MatchManager (src/server/multiplayer/matchManager.ts):
- Class that manages the lifecycle of a single match within a room
- `startCountdown(roomId)`: broadcasts "match:countdown" with {count: 3}, then 2, then 1, then "match:start" at 1-second intervals. All clients receive the same timestamps for sync.
- `eliminatePlayer(roomId, playerId, reason: "topout" | "disconnect")`: marks player as eliminated in MatchState, records elimination order (timestamp or index), broadcasts "match:elimination" to all players
- `checkWinCondition(roomId)`: after each elimination, check if only 1 player alive. If so, broadcast "match:finished" with results.
- `generateResults(roomId)`: returns array of players ranked by elimination order — winner (last alive) is rank 1, last eliminated is rank 2, etc. Include each player's final score, level, lines cleared.
- `endMatch(roomId)`: cleans up match state, transitions room back to lobby status so host can start a new match

Wire into existing game handlers (src/server/handlers/gameHandlers.ts):
- `game:topout` event: player reports their board is full, call matchManager.eliminatePlayer with reason "topout"
- On WebSocket disconnect during active match: call matchManager.eliminatePlayer with reason "disconnect" (per user decision: disconnected player is instantly eliminated)
- `game:start` from host: triggers matchManager.startCountdown
- After "match:finished" broadcast, schedule room transition back to lobby after a delay (~10 seconds for results viewing)

- Props: `results: MatchResult[]` (ranked array), `currentPlayerId: string`

Create useMultiplayerMatch hook (src/client/hooks/useMultiplayerMatch.ts):
- Manages client-side match state machine: `idle` -> `countdown` -> `playing` -> `finished`
- Listens to WebSocket events: "match:countdown", "match:start", "match:elimination", "match:finished"
- On "match:countdown": sets matchPhase to "countdown", stores count
- On "match:start": sets matchPhase to "playing", enables game input
- On "match:elimination": updates player states (gray out eliminated in UI via plan 02's MiniBoard)
- On "match:finished": sets matchPhase to "finished", stores results for MatchResults component
- Returns: `{ matchPhase, countdownNumber, results, players, isEliminated }`
- When current player is eliminated: disable game input, show "You were eliminated" overlay but keep watching
