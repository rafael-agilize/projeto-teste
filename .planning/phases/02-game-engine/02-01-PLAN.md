```typescript
// Position on the board
export interface Position { x: number; y: number; }

// Rotation states (0=spawn, 1=CW, 2=180, 3=CCW)
export type Rotation = 0 | 1 | 2 | 3;

// Direction for movement
export type Direction = 'left' | 'right' | 'down';

// Piece types
export type PieceType = 'I' | 'O' | 'T' | 'S' | 'Z' | 'J' | 'L';

// Active piece state
export interface Piece {
  type: PieceType;
  position: Position;  // top-left of bounding box
  rotation: Rotation;
}

// Cell is null (empty) or a piece type (for color)
export type Cell = PieceType | null;

// Board is 10 columns x 20 visible rows (+ 2-4 hidden rows above)
export type BoardState = Cell[][];  // [row][col], row 0 = top

// Scoring event types
export type ClearType = 'single' | 'double' | 'triple' | 'tetris';
export type ScoreEvent = ClearType | 'tspin-single' | 'tspin-double' | 'tspin-triple' | 'tspin-mini' | 'combo' | 'back-to-back';

// Full game state (serializable for multiplayer)
export interface GameState {
  board: BoardState;
  currentPiece: Piece | null;
  holdPiece: PieceType | null;
  holdUsed: boolean;  // can only hold once per piece
  nextQueue: PieceType[];  // at least 5 pieces
  score: number;
  level: number;
  linesCleared: number;
  combo: number;  // -1 = no combo, 0+ = consecutive clears
  backToBack: boolean;  // last clear was tetris or t-spin
  isGameOver: boolean;
  elapsedMs: number;
}
```

The GameState must be fully serializable (no functions, no DOM refs) for Phase 3 multiplayer sync.

  <action>
Implement the four engine modules:

**src/engine/pieces.ts:**
- Define PIECE_SHAPES: Record<PieceType, Position[][]> with all 4 rotation states for each piece per SRS specification
- PIECE_COLORS: Record<PieceType, string> with locked colors (cyan I, yellow O, purple T, blue J, orange L, green S, red Z)
- createPiece(type, startCol): creates Piece at top of board (row 0-1 depending on type)
- getPieceCells(piece): returns absolute Position[] of occupied cells for current rotation

**src/engine/board.ts:**
- createBoard(): returns 24-row x 10-col grid (rows 0-3 are buffer above visible area)
- isValidPosition(board, piece): checks all piece cells are in bounds and not occupied
- placePiece(board, piece): immutably writes piece cells onto board
- clearLines(board): finds and removes full rows, returns { board: BoardState, linesCleared: number }

**src/engine/srs.ts:**
- SRS_KICKS: wall kick offset tables for JLSTZ pieces (standard 5-test table)
- I_KICKS: wall kick offset tables for I piece (different from JLSTZ)
- O piece has no kicks (only state 0)
- tryRotation(board, piece, direction: 'cw' | 'ccw'): tries base rotation then each kick offset, returns new Piece with updated rotation+position if valid, or null if all kicks fail

**src/engine/bag.ts:**
- Uses double-bag approach: maintain array, refill with shuffled 7 when running low
- createBag(): returns initial bag state with 14 pieces (2 shuffled sets)
- nextPiece(bag): returns { piece: PieceType, bag: updated bag state }
- Fisher-Yates shuffle for randomization

All functions are pure (no mutation) for easy state serialization.
