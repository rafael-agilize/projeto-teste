Implement Canvas 2D rendering for the complete Tetris game:

**src/renderer/canvas.ts - GameRenderer class:**
- Constructor takes canvas element, computes cell size to fit the board (10x20 visible) within the canvas
- Cell size: aim for ~30px cells (board area ~300x600px), with room for side panels
- `render(state: GameState, ghostPosition: Position)`: clears canvas, draws in order:
  1. Board background: solid dark/black fill, no grid lines (per user decision)
  2. Placed pieces: filled rectangles in piece colors (solid colors, per user decision)
  3. Ghost piece: outline-only rectangles (1-2px stroke, semi-transparent, per user decision)
  4. Current piece: filled rectangles in piece color
- Each cell rendered as a solid colored square with a slight 1px darker border for definition
- Colors per user decision: cyan I, yellow O, purple T, blue J, orange L, green S, red Z

**src/renderer/hud.ts - HUDRenderer class:**
- Renders alongside the board on the same canvas (or a separate overlay canvas)
- **Left side panel:** Hold piece box
  - Label "HOLD" above
  - Shows held piece preview (or empty box if none)
  - Dim/gray when holdUsed=true (can't use again this turn)
- **Right side panel:** Next-5 queue
  - Label "NEXT" above
  - Shows 5 upcoming pieces vertically stacked
  - Each piece rendered in miniature in its correct color
- **Below or beside board:** Score display
  - Score: right-aligned number with label
  - Level: current level number
  - Lines: total lines cleared
- Use a clean monospace or pixel-style font (system monospace is fine, retro polish is Phase 3)

**src/style.css:**
- Center the canvas on the page
- Dark background for the page (match the game aesthetic)
- No scrollbars, full viewport usage
- `canvas { display: block; margin: 0 auto; image-rendering: pixelated; }`

<task type="auto">
  <name>Task 2: Effects and main game loop integration</name>
  <files>src/renderer/effects.ts, src/main.ts, index.html</files>
  <action>
Implement visual effects and wire the full game loop:

**src/renderer/effects.ts - EffectsManager class:**
- **Floating text popups:** When score events occur (tetris, T-spin, combo, back-to-back):
  - Render text on the board area (e.g., "TETRIS!", "T-SPIN DOUBLE", "COMBO x3", "BACK-TO-BACK")
  - Text floats upward and fades out over ~1 second
  - Per user decision: floating text popups on the board for scoring events
  - Use bold text, white with slight glow/shadow for readability
- **Line clear animation:**
  - Flash/white-out the cleared rows briefly (~200ms)
  - Then collapse rows above smoothly (~100ms)
  - Keep it simple but visible
- **Game over overlay:**
  - Semi-transparent dark overlay on the board
  - "GAME OVER" text centered
  - Final stats: Score, Level, Lines, Time played (formatted as mm:ss)
  - "Play Again" button (styled, clickable)
  - Per user decision: game over screen with final stats and Play Again

**src/main.ts - Main entry point:**
Wire everything together with requestAnimationFrame:

1. Get canvas from DOM
2. Create GameRenderer, HUDRenderer, EffectsManager
3. Create InputHandler
4. Create initial GameState via createGame()
5. Start game loop:
   - requestAnimationFrame callback:
     a. Calculate deltaMs since last frame
     b. Poll InputHandler for actions
     c. Apply each action to game state (moveLeft, moveRight, etc.)
     d. Call gameTick(state, deltaMs) for gravity
     e. Check for line clears then calculate score then trigger floating text
     f. Compute ghost position
     g. Render: canvas.render(state, ghost), hud.render(state), effects.update(deltaMs)
     h. If game over: show game over overlay, stop game loop
     i. If paused: skip game logic, show "PAUSED" overlay
6. "Play Again" handler: reset game state, restart loop

**Integration of scoring into game loop:**
- After placePiece + clearLines in gameTick, call calculateScore with the clear result
- Feed T-spin detection result from detectTSpin
- Update state.score, state.combo, state.backToBack, state.linesCleared, state.level
- Pass score events to EffectsManager for floating text

**Update index.html:**
- Canvas sized appropriately (e.g., 800x700 to fit board + side panels)
- Import src/main.ts as module

**Performance target:** Solid 60fps. Use requestAnimationFrame, avoid allocations in hot path where practical.
