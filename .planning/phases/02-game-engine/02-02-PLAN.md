<task type="auto" tdd="true">
  <name>Task 1: Game state machine</name>
  <files>src/engine/game.ts, src/engine/game.test.ts</files>
  <behavior>
    - createGame() returns initial GameState with empty board, first piece from bag, 5+ next queue
    - moveLeft/moveRight: shifts piece if valid, returns updated state
    - softDrop: moves piece down 1 row, awards 1 point per row
    - hardDrop: instantly drops piece to ghost position, awards 2 points per row, locks immediately
    - getGhostPosition(state): returns Position of where current piece would land (lowest valid row)
    - holdPiece: swaps current with held piece (or draws from queue if no held), sets holdUsed=true, resets on new piece
    - holdPiece returns same state if holdUsed is already true
    - gameTick(state, deltaMs): moves piece down when timer exceeds drop interval for current level
    - Lock delay: piece has 500ms after landing before locking; resets on successful move/rotate (max 15 resets)
    - On lock: place piece on board, clear lines, spawn next piece, reset holdUsed
    - Game over: if new piece spawns in an invalid position, isGameOver = true
  </behavior>
  <action>
Implement src/engine/game.ts as a pure functional state machine. Every function takes GameState and returns new GameState (no mutation).

**Game tick logic:**
- Track `dropTimer` (ms since last drop). When timer >= drop interval, move piece down
- Drop interval by level: `Math.max(1000 * Math.pow(0.8 - ((level - 1) * 0.007), level - 1), 16.67)` (capped ~60fps at high levels, cap at level 15-20 per user decision)
- Lock delay: when piece is resting on surface (can't move down), start 500ms lock timer
- If player moves/rotates successfully while on surface, reset lock timer (up to 15 resets max)
- When lock timer expires: call placePiece, clearLines, spawn next from queue

**Ghost piece:**
- getGhostPosition: clone current piece, move down repeatedly until invalid, return last valid position
- Ghost is computed on every state change (not stored in state -- derived)

**Hold piece:**
- First hold: store current piece type, draw next from queue
- Subsequent hold: swap current piece type with held type
- Reset position to spawn position on swap
- holdUsed flag prevents multiple holds per piece turn

**Game over detection:**
- After spawning new piece, check isValidPosition
- If invalid at spawn position, set isGameOver = true

All state transitions return new objects. GameState remains serializable.

  <name>Task 2: Keyboard input handler with DAS/ARR</name>

  <action>
Create InputHandler class that maps keyboard events to game actions with proper auto-repeat:

**Key bindings:**
- Left Arrow / A: move left
- Right Arrow / D: move right
- Down Arrow / S: soft drop
- Up Arrow / W: rotate CW
- Z: rotate CCW
- Space: hard drop
- Shift / C: hold piece
- Escape / P: pause (toggle)

**DAS/ARR (Delayed Auto Shift / Auto Repeat Rate):**
- DAS: 133ms (initial delay before auto-repeat starts)
- ARR: 10ms (repeat interval once auto-repeat is active) -- fast enough to feel responsive
- Only applies to left/right movement
- Soft drop has its own repeat: immediate repeat at ~50ms

**Implementation:**
- Listen to keydown/keyup on document
- Track pressed keys and timestamps
- Expose `getActions(currentTimeMs): GameAction[]` method polled by game loop
- GameAction = 'moveLeft' | 'moveRight' | 'softDrop' | 'hardDrop' | 'rotateCW' | 'rotateCCW' | 'hold' | 'pause'
- Hard drop, rotate, and hold are single-fire (trigger once per keypress, not on hold)
- Prevent default on game keys to avoid page scrolling
- destroy() method to remove event listeners
