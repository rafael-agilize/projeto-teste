Create src/lib/types.ts — shared types:
```typescript
export interface Player {
  id: string;          // socket.id
  nickname: string;
  isHost: boolean;
  isReady: boolean;
}

export interface Room {
  code: string;        // 4 uppercase letters
  players: Player[];
  maxPlayers: 4;
  status: 'waiting' | 'countdown' | 'playing';
  hostId: string;      // socket.id of host
}

// Socket events
export interface ServerToClientEvents {
  'room:created': (data: { code: string }) => void;
  'room:joined': (data: { room: Room }) => void;
  'room:player-joined': (data: { player: Player }) => void;
  'room:player-left': (data: { playerId: string }) => void;
  'room:updated': (data: { room: Room }) => void;
  'room:error': (data: { message: string }) => void;
  'room:countdown': (data: { count: number }) => void;
  'room:game-start': () => void;
}

export interface ClientToServerEvents {
  'room:create': (data: { nickname: string }) => void;
  'room:join': (data: { code: string; nickname: string }) => void;
  'room:ready': (data: { ready: boolean }) => void;
  'room:start': () => void;
}
```

Create src/lib/room.ts — server-side room management:
- In-memory Map<string, Room> storage
- `generateCode()`: random 4 uppercase letters, retry if collision
- `createRoom(socketId: string, nickname: string): Room` — creates room, adds player as host
- `joinRoom(code: string, socketId: string, nickname: string): Room | { error: string }` — validates: room exists (else "Room not found"), not full (else "Room is full"), returns updated room
- `leaveRoom(socketId: string): { room: Room | null; deleted: boolean }` — removes player, transfers host to next player if host left, deletes room if empty
- `getRoom(code: string): Room | undefined`
- `setPlayerReady(socketId: string, ready: boolean): Room | null`
- `canStart(code: string): boolean` — returns true if 2+ players are ready
- `getRoomByPlayerId(socketId: string): Room | undefined` — find room containing player

Create src/server.ts — custom Next.js server with Socket.io:
- Use `next` and `http` modules to create a custom server
- Attach Socket.io server to the HTTP server
- Socket event handlers:
  - 'room:create': call createRoom, join socket to room channel, emit 'room:created' + 'room:joined'
  - 'room:join': call joinRoom, handle errors (emit 'room:error'), join socket to room channel, emit 'room:joined' to joiner + 'room:player-joined' to room
  - 'room:ready': call setPlayerReady, emit 'room:updated' to room
  - 'room:start': verify caller is host + canStart, emit countdown sequence (3,2,1 with 1s delays), then 'room:game-start'
  - 'disconnect': call leaveRoom, emit 'room:player-left' + 'room:updated' to room (including host transfer info)
- Update package.json "dev" script to use `ts-node --esm src/server.ts` or `tsx src/server.ts`. Install tsx as dev dep: `npm install -D tsx`
- Add a "dev:custom" script: `tsx watch src/server.ts`

Create src/lib/socket.ts — client singleton:
- Export a function `getSocket()` that returns a singleton socket.io-client instance
- Connect to `window.location.origin` (same server)
- Export the socket instance for use in components
